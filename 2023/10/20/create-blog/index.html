6、issue：

        在平板或者手机上有时会出现输入url订阅链接依然翻不了墙的情况，包括在电脑上有的时候导入订阅文件之后点击导入的链接会显示error，表示Switch不了订阅文件。以前不懂，现在懂了，就是订阅文件里面有的节点有错误，这个时候就得先在电脑上的clash导入，然后点击切换，看能不能成功，不能成功就得根据提示的错误删除对应的节点，一般情况下删除一个节点要对应删除两个位置，一个是error的地方，一个就是下面的proxy对应的节点。这是因为clash只支持wmess协议，不支持vless协议，proxy n的n跟数组下标一样从0开始。

        上面说的都是电脑端的处理方式，而手机端或者ipad上，在连接了url后发现不能翻墙，就得根据电脑上查阅出来、定位error的地方，把对应的几个地方的节点直接删除就了。

一、引言

1、我为什么搭建博客

        大概是从什么时候开始有了要搭建一个博客的想法呢？比较强烈的时候应该是大三下这段时间吧，其实之前就有过这种想法，只是一直没有这么强烈的想。

        至于原因，其一大概是大三下因为保研、实习种种因素，刷题或者各种材料都挺杂乱的，就想着做个博客吧，顺便也能记录下学习路线和知识点；其二，因为看过许多优秀学长学姐的博客网站，自己也想搭建一个，看一下这种都是怎么弄的，也算拓展下自己的知识和技能；其三，因为K的关系，我偶尔也想记录下他不在身边的这些年，或者想记录心情与生活细节，毕竟现生中越来越少会有人愿意倾诉你的事情，大家都很忙，那你总得需要一个角落或发泄或分享。也算是受身边环境的影响吧，我的大学生活说实话都很枯燥，人一无聊就会瞎想，偶尔复习周或者课业比较忙的时候还好一点。

        最后主要还是保研后确实时间空余很多。

2、常见的博客框架

        对于学生而言常用来搭建博客的框架有hexo、WordPress、typecho + github +（(云)服务器）。

        hexo我看到的比较多的都是直接结合github pages直接建的静态网页，挺吸引人的，界面简洁、维护简单、免费，见过的很多学生博客也大都是这种搭建方式。但是问题是写博客、发表博客程序有些冗杂…而且，其实对于我来说这种方式不太适用，因为我应该是需要写很多仅自己可见的博客的，但是hexo不支持这种设置，WordPress支持。而且hexo是完全静态的，只能本地编写然后hexo部署到github上，WordPress和typecho是动态的，也就是可以在浏览器即写即发布，仪表盘直接管理设置也很方便…不过缺点就是收费。

基于Serverless搭建WordPress个人博客【弃稿】

一、基础知识

1、WP的部署

        WordPress搭建博客有两种部署方式，可以使用腾讯云的Serverless服务部署，也可以部署到github pages进行静态部署。

2、Serverless（无服务器）

        Serverless Framework 无服务器应用框架，SFC 是腾讯云云函数，是腾讯云提供的无服务器执行环境。serverless 只是一个定义，SFC 是其的实现。

        使用传统方式部署应用时首先要购买云服务器，再在服务器上进行环境配置，然后再去部署我们的应用代码，而用Serverless部署应用不需要购买服务器。

        无服务器不是没有服务器，因为开发应用不可能没有服务器，Serverless 只是表示当使用 Serverless 时，不需要关心底层资源，不需登录和优化服务器，只需关注最核心的代码片段。当使用 Serverless 部署应用时，不用每次都去部署开发环境，可以大大节省时间。

3、WordPress介绍

        WordPress 是基于 PHP 和 MySQL 的免费开源内容管理系统（CMS）。WordPress 始于 2003 年，最开始仅为一款简单的博客系统，但现已发展成为具有数千款插件，小工具和主题功能完整的CMS系统。

四、Serverless应用监控

        Serverless 为开发者提供了应用监控平台，监控数据都可以进行可视化分析。
